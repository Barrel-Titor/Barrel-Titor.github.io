用 Hexo 和 Github Page 建立简单的 Blog

每次折腾点新事物，都是疯狂踩坑的过程，但是也从中学到了不少。
用博客做点记录，方便以后同样踩坑的朋友解决问题，也方便自己日后回顾。

1 - 为什么选择 Hexo 和 Github Page

相信有心写博客的新手，一开始都会纠结选择哪个平台。就好像我打游戏建号的时候，纠结选用什么武器，最后总会发现，这 TM 根本不是重点。

1.1 - Github Page

对于多个平台的优缺点分析，有大佬已经做过了，有兴趣的朋友请自行移步。

我个人选用 Github Page 的理由：

  1. 不会有奇奇怪怪的审核君；
  2. 学习、熟悉一下 Web 开发和 Github 的使用
  3. 本身比较熟悉 Markdown 语法，撰写方便
  4. 免费QAQ

1.2 - Jekyll or Hexo?

在 Github Page 上有两个主流的静态博客框架：Jekyll 和 Hexo。Jekyll 与 Github 契合度更高，可以在 Github 中直接生成页面；Hexo 中文资料更丰富，基础功能更多，但是部署和备份比 Jekyll 麻烦。

Jekyll Hexo
页面生成 Github 直接生成静态页面 本地生成静态页面，然后上传
分支管理 一个分支即可 推荐使用两个分支，一个设置文件，一个页面文件

两者我都有浅尝，目前选用 Hexo，因为其自带了博客标签和分类的功能，而 Jekyll 需要自己设置 CSS 或者用插件实现。
